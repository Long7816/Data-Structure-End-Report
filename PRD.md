# 產品需求文件 (Product Requirements Document, PRD)

## 專案名稱
**幸福通勤導航：結合下雨/高溫/YouBike 的校園動態路徑規劃系統** (Happy Commute Navigation System)

---

## 1. 專案概述 (Project Overview)
本專案為校園動態路徑規劃系統的前端展示 Demo，專為解決學生及教職員在不同氣候與交通情境下的通勤痛點（如：暴雨淋濕、烈日曝曬、陡坡階梯、YouBike 無車可借或無位可還）而設計。

本系統為 **純前端、無後端、無資料庫** 的靜態網頁應用，可直接雙擊 `index.html` 在瀏覽器中開啟運行。核心目標是透過視覺化地圖，動態展示資料結構（Graph, Hash Map）與路權加權演算法（Dijkstra / A*）在真實生活場景中的應用效果。

## 3. 功能需求 (Functional Requirements)

### 3.1 校園與周邊導航路網 (Campus Graph)
*   **路網節點 (Nodes)**：
    *   **起點**：捷運公館站、校門口、男宿、女宿等。
    *   **終點**：各棟主要教學大樓（如：綜合三館、科研大樓、教學大樓等）及特定教室。
    *   **轉乘點**：配置至少 3~5 個校園周邊與內部的 YouBike 實際站點。
*   **路網邊 (Edges)**：
    *   連接節點的通路，具備物理距離、路段類型（如：露天、有遮雨棚、樹蔭步道、樓梯、斜坡、柏油路、大理石路面等）屬性。

### 3.2 環境動態障礙點與臨界預警 (Hazard Points & Alerts)
*   **YouBike 狀態動態監控**：
    *   模擬 YouBike 站點之「可用車輛」與「可用車位」數量。
    *   **臨界值預警**：當車輛或車位 **< 3** 時，該站點節點在 UI 上標示為「黃/紅燈高風險點」。
    *   **動態避險**：若導航模式需要使用 YouBike，演算法需自動繞開高風險站點，或引導使用者至鄰近有車/有位的備用站點。
*   **路面打滑與坡度障礙**：
    *   **易打滑路段**：雨天情境下，大理石人行道、斑馬線等邊（Edge）的滑溜處罰權重提高。
    *   **陡坡障礙**：高溫情境下，爆汗陡坡處罰權重提高；無障礙模式下，陡坡與樓梯直接設為不可通行（權重設為無限大）。

### 3.3 動態環境影響區域 (Environmental Risk Areas)
*   **高溫曝曬區 (Heat Exposure Zones)**：
    *   UI 畫面上需能框出「無遮蔽的正午曝曬陽光區」，在此區域內的邊（Edge）於「最涼模式」下會被施加高額處罰權重。
*   **大雨傾盆區 (Heavy Rain Zones)**：
    *   標示出缺乏騎樓、連通道的露天淋雨路段，在此區域內的邊於「少淋雨模式」下會被施加高額處罰權重。

### 3.4 幸福通勤最佳導航路線 (Navigation Routing)
*   **多模式即時切換**：系統需提供至少 4 種路由模式：
    1.  **最快路線 (Fastest)**：物理距離最短（純 Dijkstra / A*，不考慮環境懲罰）。
    2.  **最涼路線 (Coolest)**：避開曝曬區，優先行駛/步行於樹蔭、遮蔽物下。
    3.  **少淋雨路線 (Rain-avoiding)**：雨天情境下，避開露天淋雨及打滑路段，優先走走廊、連通道。
    4.  **無障礙路線 (Accessible)**：完全過濾樓梯與高難度陡坡，僅通行平地、斜坡或設有電梯的通道。
*   **雙路線視覺化對比**：
    *   地圖上必須同時渲染 **「原本的最快路線（灰色/虛線）」** 與 **「優化後的幸福路線（彩色/實線）」**，以便直觀對比演算法在不同環境因子下的調整成效。

## 4. 技術架構與資料結構 (Technical Architecture & Data Structures)

### 4.1 系統架構
本系統為單頁 Web 應用程式 (SPA)，架構如下：
```
├── index.html            # 網頁結構與 UI 佈局
├── style.css             # 現代感視覺設計 (Dark Mode, 漸層, 玻璃擬態)
├── script.js             # 應用程式主邏輯、地圖渲染與路徑規劃演算法
└── data/
    ├── campusGraph.js    # 靜態地圖路網資料 (Nodes & Edges 鄰接清單)
    └── disasterScenario.js  # 模擬天氣情境與 YouBike 動態測資 JSON
```

### 4.2 核心資料結構
1.  **Graph (圖形結構)**：
    *   儲存於 `campusGraph.js`。
    *   採用 **鄰接清單 (Adjacency List)** 格式，便於 Dijkstra / A* 演算法進行節點擴展與鬆弛 (Relaxation)。
2.  **Hash Map (雜湊表)**：
    *   在 JavaScript 中以 `Map` 或 `Object` 實現。
    *   用於以 $O(1)$ 時間複雜度動態查詢與更新：
        *   YouBike 站點即時車位狀態（`stationId -> { bikes, spaces }`）。
        *   動態環境因子懲罰權重（`edgeType -> penaltyMultiplier`）。

### 4.3 演算法機制 (Penalty Mechanism)
*   邊的實際計算權重公式為：
    $$\text{EffectiveWeight} = \text{PhysicalDistance} \times (1 + \sum \text{Penalties})$$
*   不同模式下的 Penalty 設定：
    *   **最快模式**：所有 Penalty 乘數皆為 0。
    *   **最涼模式**：曝曬區路段之 Penalty $\ge 5.0$，樹蔭路段 Penalty = 0。
    *   **少淋雨模式**：露天無遮蔽路段之 Penalty $\ge 5.0$，易打滑路段 Penalty $\ge 3.0$。
    *   **無障礙模式**：樓梯與陡坡路段之 Penalty = $\infty$ (禁止通行)。

## 5. UI/UX 與簡報展示設計 (Presentation-Ready UI)

*   **現代化美學設計**：
    *   採用深色調主題 (Dark Mode) 或精緻的高對比色彩搭配。
    *   使用 Google Fonts (如 Outfit 或 Inter) 提升易讀性。
    *   卡片與控制面板採用玻璃擬態 (Glassmorphic) 效果，搭配平滑的微動畫 (Micro-animations)。
*   **直覺式側邊控制面板 (Sidebar Control Panel)**：
    *   **起終點選擇器**：下拉選單或地圖直覺點選。
    *   **情境觸發器 (Scenario Trigger)**：快速一鍵切換天氣情境（如：「烈日正午」、「突發暴雨」、「YouBike滿位警告」）。
    *   **模式切換鈕**：最快、最涼、少淋雨、無障礙。
*   **地圖視覺化區域 (Map Canvas/SVG)**：
    *   以 SVG 或 Canvas 繪製簡潔質感的校園地圖。
    *   高溫曝曬區以半透明暖橘色覆蓋；大雨區以藍灰色雨絲特效或半透明藍色框線表示。
    *   YouBike 站點依狀態亮起綠燈（充足）、黃燈（警戒 $<3$）、紅燈（無車/無位）。

## 6. 驗收標準與 Demo 流程 (Acceptance Criteria & Demo Flow)

### 6.1 基本執行與相容性
*   **零部署要求**：不需安裝 Node.js、npm 或任何伺服器，直接雙擊 `index.html` 即可在各大主流瀏覽器中正常運作。
*   **主控台無錯**：網頁載入與操作期間，瀏覽器 Console 不得出現任何 JS 報錯。
*   **防呆機制**：若未選擇起終點即點選導航，系統應跳出直覺提示而非當機。

### 6.2 3分鐘黃金 Demo 流程 (3-Minute Presentation Path)
1.  **Step 1：基本導航展示**
    *   選擇起點「捷運公館站」，終點「科研大樓」，選擇「最快路線」。
    *   系統即時繪製出一條沿著公館外圍穿越廣場的最短灰色虛線。
2.  **Step 2：情境一「突發暴雨」**
    *   點擊「突發暴雨」情境，地圖特定區域變為藍色雨區，YouBike 站點狀態隨之更新。
    *   切換為「少淋雨路線」。
    *   系統即時更新路線（彩色實線），自動繞開露天廣場，引導使用者走行政大樓的連通道與騎樓，同時顯示原本的灰色虛線路徑以資對比。
3.  **Step 3：情境二「YouBike 站點滿位/無車」**
    *   展示某 YouBike 站點亮紅燈（可用車 $< 3$），演算法在計算「少淋雨 + YouBike」轉乘時，會自動避開該站，改為引導至鄰近有車的備用 YouBike 站。
4.  **Step 4：說明與結束**
    *   口頭或畫面上簡述 Hash Map 更新權重的 $O(1)$ 特性與 Dijkstra / A* 邊權重處罰機制。
