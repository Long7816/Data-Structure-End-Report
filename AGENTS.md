## Project Overview
  本專案是「智行北科 (OmniNTUT)：結合下雨/高溫/空污/YouBike 的校園動態路徑規劃系統」的前端展示 Demo。
  目標是做出一個可以在瀏覽器中打開的靜態網站，展示：
  •	校園與周邊地圖節點： 包含捷運站、宿舍、各棟教學大樓與教室。
  •	YouBike 站點動態監控： 串接臺北市開放資料，除了即時車輛與車位狀態外，特別針對騎士最怕的「無車可借」與「滿位無法還」設計動態臨界值預警（如可用數 < 3 則判定為高風險節點）。
  •	動態環境變數區域： 包含降雨影響與易打滑路段、高溫曝曬與樹蔭遮蔽區域。
  •	幸福通勤最佳導航路線： UI 可即時切換「最快 / 最涼 / 少淋雨 / 無障礙」等 4 種模式。
  •	資料結構與演算法實作： 展示 Dijkstra、 A* 演算法與 Hash Map 如何高效處理路權權重（如逆風、坡度、曝曬處處罰權重）與動態資料查詢。

##Must Have
  1. 校園與周邊導航地圖節點 (Campus & Surrounding Graph)
    •	捷運/宿舍到教室的完整路網： 地圖上必須清晰標示出起點（如捷運公館站、校門口、男/女宿）與終點（各棟教學大樓、特定教室）的節點。
    •	YouBike 站點配置： 地圖上需包含至少 3~5 個校園周邊與內部的 YouBike 實際站點位置作為特殊圖層。
  2. 環境動態障礙點與臨界預警 (Environmental "Hazard" Points)
    •	YouBike 站點斷貨/滿位預警（騎士痛點）： 當特定 YouBike 站點可用車輛或空位 < 3 時，在畫面上將該節點標示為「黃/紅燈危險點」，演算法需具備自動繞道或導向鄰近備用站點的功能。
    •	路面打滑與坡度障礙： 雨天時的易打滑材質路段（如大理石人行道、斑馬線），或高溫時的爆汗坡道路段。
  3. 動態環境影響區域 (Dynamic Risk Areas)
    •	高溫曝曬： 網頁畫面上需能框出「無遮蔽的正午曝曬陽光區」。
    •	大雨傾盆區域： 標示出無遮蔽物、缺乏騎樓或校園連通道的露天淋雨路段。
  4. 幸福通勤最佳導航路線 (Optimal Commute Paths)
    •	多模式即時切換： 系統必須能根據 UI 切換，渲染出至少 3 種截然不同的路線：
      o	「最快路線」： 物理距離最短（純 Dijkstra / A* 計算）。
      o	「最涼/少淋雨路線」： 自動繞開曝曬/淋雨區，優先走樹蔭與走廊。
      o	「無障礙路線」： 繞開樓梯高坡。
  5. 資料結構與演算法圖形說明 (Data Structure & Algorithm Explanation)
    •	Hash Map 的動態權重應用： 說明如何利用 Hash Map 以 $O(1)$ 的時間複雜度，即時查詢與更新 YouBike 站點車位狀態及環境變數。
    •	Dijkstra & A* 的路權處罰（Penalty）機制： 清楚說明演算法如何動態調高（處罰）惡劣環境路段的邊權重（Edge Weight），而非單純計算物理距離。
  6. 適合期末簡報展示的網頁畫面 (Presentation-Ready UI)
    •	直覺的側邊控制面板： 包含 5 種模式的切換按鈕（最快 / 最涼 / 少淋雨 / 避空污 / 無障礙）。
    •	清晰的視覺化對比： 當使用者切換模式時，地圖上要能同時或用不同顏色呈現「原本的最快路線」與「優化後的幸福路線」，讓評審老師一眼看出演算法的實作成果（極具 Scannability）。

    o	使用者能在 3 分鐘內流暢完成操作：進入首頁  觸發特定天氣情境  點擊切換模式  系統秒級更新並渲染結果 。
  •	具備明確的「輸入、處理、輸出」資料流： 
    o	輸入 (Input)： 使用者能在 UI 點選起點、終點，或透過按鈕觸發本地模擬的環境變數 JSON（如：突然暴雨、AQI 超標、特定站點 YouBike 滿位）。
    o	處理 (Process)： 系統讀取 environmentCondition.js，透過 Hash Map 動態更新圖形邊權重，並由 Dijkstra 或 $A^*$ 演算法重新計算路徑 。
    o	輸出 (Output)： 網頁地圖上即時用不同顏色渲染出兩條對比路線（如：優化前的最快路線與優化後的幸福路線），提供極佳的視覺化簡報對比 。
  •	核心資料結構與演算法實作驗證：
    o	Graph (圖形結構) 應用： 校園與周邊路網必須確實以節點（Nodes）與邊（Edges）的鄰接清單/矩陣形式存儲於 campusGraph.js 中 。
    o	Hash Map (雜湊表) 應用： YouBike 站點車位狀態（可用車、可用位）與環境變數必須利用 Hash Map 做到 $O(1)$ 的動態查詢與權重覆蓋 。
    o	演算法實作： 「最涼/少淋雨/避空污/無障礙」模式不能只是靜態切換文字，必須有實際透過 Penalty（權重處罰）機制改變邊權重（Edge Weight）而算出不同路線的程式碼運作結果 。
  •	邊界條件與系統穩定性：
    o	YouBike 動態臨界值： 當目的地站點之車位或借車點之可用車 $< 3$ 時，地圖節點必須呈現紅/黃燈警告，且演算法能成功自動繞道或導向鄰近備用站點 。
    o	防呆機制： 在使用者尚未選定起訖點或環境資料即盲目點擊「開始導航」時，系統應跳出直覺的 UI 提示，而不是產生 JavaScript 錯誤或導致畫面凍結 。
  •	補充自動化測試驗收：以上驗收與 Demo 流程，建議可使用 browser use 功能 (/browser) 錄製或跑過至少 3 次主要路徑測試，確保在簡報上台前沒有任何中斷 demo 的致命 Bug 。
